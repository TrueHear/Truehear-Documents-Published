Authentication (Customer)

Create a new customer user (tenant/provider/site scoped)

Creates a new customer user within a specific tenant, provider, or site context. This endpoint enforces hierarchical validation and membership-based RBAC, allowing only authorized customer administrators to create users within their scope. The endpoint automatically: - Validates tenant → provider → site relationships. - Generates a strong temporary password (visible once). - Creates linked User, Credential, and Membership records. - Assigns the specified roles (or defaults to provider_staff).

Authorizations:
BearerAuth
Request Body schema: application/json
required
tenantId
required
string

The unique identifier of the tenant (organization) to which the user belongs.

providerId
string or null

The provider (hospital or clinic) identifier under the tenant.

siteId
string or null

Optional site identifier under the provider, for granular access scoping.

email
required
string <email>

The new user's email address (must be unique across the tenant).

firstName
string

The user's first name.

lastName
string

The user's last name.

phone
string

Optional contact phone number.

username
string

Optional username for login (if different from email).

roles
Array of strings
Items Enum: "tenant_admin" "tenant_manager" "tenant_hr_manager" "tenant_it_manager" "tenant_compliance" "provider_admin" "provider_manager" "provider_clinician" "provider_staff"

List of role keys to assign to this user. If not specified, defaults to ["provider_staff"].

Responses

Response Schema: application/json
status
boolean
message
string
object

Development

http://localhost:8888/api/auth/customers/user/create

Production

https://localhost:8888/api/auth/customers/user/create
